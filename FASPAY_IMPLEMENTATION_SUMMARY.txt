📊 DATABASE SCHEMA
═══════════════════════════════════════════════════════════════════════════════

Table: faspay_test_transactions

Columns:
  • id - Primary key
  • user_id - Foreign key to users table
  • bill_no - Unique order number (indexed)
  • bill_description - Order description
  • amount - Transaction amount
  • currency - Currency code (default: IDR)
  • customer_name - Customer full name
  • customer_email - Customer email
  • customer_phone - Customer phone
  • status - Payment status (unpaid|processing|paid|failed|expired|cancelled)
  • trx_id - Faspay transaction ID (indexed)
  • payment_reff - Payment reference from Faspay
  • payment_channel - Payment method used
  • payment_date - Timestamp when paid
  • bank_user_name - Bank/channel user name
  • payment_response - Full webhook response (JSON)
  • notes - Internal notes
  • metadata - Additional data (JSON)
  • created_at - Created timestamp (indexed)
  • expires_at - Expiration timestamp
  • updated_at - Updated timestamp

🚀 QUICK START
═══════════════════════════════════════════════════════════════════════════════

1. CONFIGURE CREDENTIALS
   Edit .env and add:
   FASPAY_MERCHANT_ID=your_merchant_id
   FASPAY_USER_ID=your_user_id
   FASPAY_PASSWORD=your_password
   FASPAY_API_KEY=your_api_key

2. RUN MIGRATION
   php artisan migrate

3. REGISTER WEBHOOKS
   Go to Faspay merchant dashboard:
   Settings → Webhook Configuration
   
   Add:
   Notification: https://survey.ddev.site/api/webhook/faspay/notification
   Return: https://survey.ddev.site/transaction/faspay/return

4. TEST
   Login and go to: https://survey.ddev.site/faspay/test/transactions

🔐 PAYMENT FLOW
═══════════════════════════════════════════════════════════════════════════════

1. User creates test transaction
   ↓
2. Transaction saved to database
   ↓
3. User redirected to payment page
   ↓
4. User clicks "Proceed to Payment"
   ↓
5. FaspayService::createInvoice() called
   ↓
6. User redirected to Faspay payment gateway
   ↓
7. Customer completes payment on Faspay
   ↓
8. Faspay redirects to return URL (callback)
   ↓
9. Payment notification sent to webhook
   ↓
10. FaspayController::notification() validates & processes
   ↓
11. Transaction status updated to 'paid'
   ↓
12. User sees success page

         Check Faspay configuration status

🔒 SECURITY FEATURES
═══════════════════════════════════════════════════════════════════════════════

1. SIGNATURE VALIDATION
   Every webhook is validated using:
   SHA1(MD5(user_id + password + bill_no + payment_status_code))
   
2. CSRF TOKEN BYPASS
   Webhook endpoint bypasses CSRF token check via:
   withoutMiddleware(VerifyCsrfToken::class)
   
3. CREDENTIAL STORAGE
   All sensitive data stored in .env (not in code)
   Never commit credentials to repository
   
4. AUDIT LOGGING
   All payment operations logged with timestamp
   Logs stored in: storage/logs/laravel.log

📞 SUPPORT & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Common Issues:

1. "Faspay is not configured"
   → Check .env for FASPAY_MERCHANT_ID, USER_ID, PASSWORD
   → Run: php artisan config:cache

2. "Signature validation failed"
   → Verify credentials match Faspay dashboard
   → Check server time synchronization
   → Review logs: storage/logs/laravel.log

3. "Webhook not received"
   → Verify webhook URL registered in Faspay dashboard
   → Check CSRF middleware bypass
   → Test manually with curl

4. "Transaction not found"
   → Verify bill_no exists in database
   → Check if transaction hasn't expired

For detailed troubleshooting, see: FASPAY_INTEGRATION.md
